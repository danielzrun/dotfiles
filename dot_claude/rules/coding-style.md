## 🎨 Code & Comments Standards

### Code Style
- **Consistency:** Maintain consistent indentation and formatting following the specific language style guide (e.g., Prettier, gofmt).
- **Formatting:** Auto-format files where possible.

### Comments
- **Language:** Use **English** for all code comments.
- **Length Rule:** If a comment is longer than **50 characters**, place it on a separate line **above** the code, not inline.
- **Clarity:** Explain "Why" not "What".
